# 🎓 إنجاز — لوحة متابعة الدراسة اليومية

موقع شخصي (بدون أي سيرفر أو تكلفة) لمتابعة وقت قراءتك واستراحتك ونومك وإنجازاتك اليومية، مع لوحة تحكم خاصة بيك ولوحة متابعة تشاركها مع أهلك عبر رابط — تتحدّث عندهم لحظياً بفضل Firebase.

---

## 📦 شنو بيه المشروع

```
study-tracker/
├── index.html          ← لوحة المتابعة (الرابط اللي تشاركه مع أهلك)
├── admin.html           ← لوحة التحكم (خاصة بيك، لا تشاركها)
├── css/
│   ├── themes.css        ← الأنماط الأربعة (ليلي / دراسي / طبيعي / بكيفك)
│   └── style.css          ← كل التصميم والحركات
├── js/
│   ├── icons.js            ← الأيقونات
│   ├── shared.js            ← البيانات والمزامنة والأدوات المشتركة
│   ├── firebase-bridge.js    ← يربط الموقع بقاعدة بيانات Firebase (مزامنة لحظية)
│   ├── admin.js                ← منطق لوحة التحكم
│   └── viewer.js                 ← منطق لوحة المتابعة
└── README.md              ← هذا الملف
```

لا يحتاج المشروع أي تثبيت أو بناء (build) — كله ملفات HTML/CSS/JS جاهزة تشتغل مباشرة.

## 🚀 خطوات الرفع على GitHub Pages

1. **أنشئ حساب GitHub** إذا ما عندك وحد، من [github.com](https://github.com).
2. اضغط **New repository** وسمّي المستودع أي اسم (مثلاً `study-tracker`)، خلّه **Public**، ولا تفعّل أي خيار إضافي (README/gitignore) عشان يصير فاضي. اضغط **Create repository**.
3. ارفع الملفات: افتح المستودع، اضغط **Add file → Upload files**، واسحب **كل محتويات** مجلد `study-tracker` (وليس المجلد نفسه — يعني `admin.html` و`index.html` و`css` و`js` كلها مباشرة بجذر المستودع). اضغط **Commit changes**.
   - إذا تفضّل سطر الأوامر: `git init`، `git add .`، `git commit -m "أول نسخة"`، وبعدين اربطه بمستودعك وادفعه (`git push`).
4. فعّل GitHub Pages: من إعدادات المستودع **Settings → Pages**، تحت **Build and deployment** اختر **Deploy from a branch** (⚠️ **مو** "GitHub Actions" — هذا المشروع ملفات جاهزة بلا أي بناء، فلا يحتاج أي Workflow)، والفرع **main** والمجلد **/ (root)**، ثم **Save**.
5. انتظر دقيقة أو دقيقتين، وراح يطلعلك رابط شبيه بـ:
   `https://username.github.io/study-tracker/`
   هذا رابط **لوحة المتابعة** (نفس محتوى `index.html`).
   لوحة تحكمك تكون على: `https://username.github.io/study-tracker/admin.html`

> 📌 احتفظ برابط `admin.html` لنفسك بس، وشارك رابط الصفحة الرئيسية (`index.html`) مع أهلك.

## 👨‍👩‍👧 تفعيل المشاركة المباشرة مع العائلة (عبر Firebase)

بدون هذي الخطوة، الموقع يشتغل بشكل كامل بس البيانات تضل محفوظة **بجهازك بس**. المشاركة اللحظية مع العائلة تحتاج قاعدة بيانات مجانية بسيطة من Google اسمها **Firebase** — إعداد لمرة وحدة بس، وبعدها كل تحديث يوصل لأهلك **فوراً** (مو بعد دقايق، حرفياً بلحظته).

### الخطوة 1 — أنشئ مشروع Firebase
1. روح لـ [console.firebase.google.com](https://console.firebase.google.com) وسجّل دخولك بحساب Google (نفسه Gmail).
2. اضغط **Add project** (أو Create a project)، اكتب اسم أي شي (مثلاً `injaz-app`)، وكمّل الخطوات (تكدر تلغي Google Analytics، مو ضروري).

### الخطوة 2 — فعّل قاعدة البيانات والدخول المجهول
1. من القائمة الجانبية، روح لـ **Build → Realtime Database** (أو Databases & Storage → Realtime Database).
2. اضغط **Create database**، اختار أي موقع سيرفر قريب، واختار وضع البداية **Start in test mode** مؤقتاً (راح نبدّل القواعد بالخطوة الجاية على أي حال).
3. بعد ما تنشئ القاعدة، روح لتبويب **Rules** بنفس الصفحة، امسح المحتوى الموجود، والصق هذا بدله:
```json
{
  "rules": {
    "injaz": {
      ".read": true,
      ".write": "auth != null"
    }
  }
}
```
اضغط **Publish**. هذي القواعد تخلي أي شخص يقدر **يقرأ** بياناتك (عشان لوحة المتابعة تشتغل بدون تسجيل دخول)، بس **الكتابة** ما تصير إلا من جهاز مسجّل دخول — وهذا التسجيل تسويه بس لوحة التحكم تلقائياً بالخلفية (ما تحتاج كتابة إيميل أو رمز).
4. فعّل نوع الدخول: روح لـ **Build → Authentication**، اضغط **Get started**، من تبويب **Sign-in method** اختار **Anonymous** وفعّلها (Enable) ثم **Save**.

### الخطوة 3 — سجّل تطبيق ويب واحصل على الإعدادات
1. من الصفحة الرئيسية للمشروع (⚙️ Project settings)، تحت **Your apps**، اضغط أيقونة **</>）(Web).
2. سمّي التطبيق أي اسم واضغط **Register app**. ما تحتاج Firebase Hosting.
3. راح يطلعلك كود شبيه بهذا — انسخه كامل:
```js
const firebaseConfig = {
  apiKey: "AIzaSy...",
  authDomain: "injaz-app.firebaseapp.com",
  databaseURL: "https://injaz-app-default-rtdb.firebaseio.com",
  projectId: "injaz-app",
  storageBucket: "injaz-app.appspot.com",
  messagingSenderId: "...",
  appId: "..."
};
```

### الخطوة 4 — الصق الإعدادات بملفاتك
1. افتح `admin.html` بأي محرر نصوص (حتى تطبيق الملاحظات بالموبايل يشتغل)، ودوّر على السطر:
   `window.INJAZ_FIREBASE_CONFIG = null;`
2. بدّله بالكود اللي نسخته بالخطوة الماضية (خلي الاسم `INJAZ_FIREBASE_CONFIG` زي ما هو، بس عبّي القيم):
```js
window.INJAZ_FIREBASE_CONFIG = {
  apiKey: "AIzaSy...",
  authDomain: "injaz-app.firebaseapp.com",
  databaseURL: "https://injaz-app-default-rtdb.firebaseio.com",
  projectId: "injaz-app",
  storageBucket: "injaz-app.appspot.com",
  messagingSenderId: "...",
  appId: "..."
};
```
3. سوي **نفس الشي بالضبط** بملف `index.html` (نفس القيم حرفياً بالملفين).
4. ارفع الملفين المعدّلين على GitHub (Add file → Upload files، اختار الملفين، Commit changes) — هذا يحدّث النسخة المنشورة.

### الخطوة 5 — جرب المزامنة
1. افتح `admin.html`، اضغط ⚙️ **الإعدادات** → **«المشاركة مع العائلة»**.
2. اضغط **«فحص الاتصال»** للتأكد كل شي تمام (قاعدة البيانات + الدخول المجهول شغالين).
3. اضغط **«مزامنة الآن»** لنشر أول نسخة من بياناتك.
4. انسخ **رابط لوحة المتابعة** وشاركه مع أهلك — أي تحديث تسويه من هسه يوصلهم فوراً بدون ما يحتاجون يحدّثون الصفحة.

### ملاحظات مهمة عن الخصوصية والأمان
- إعدادات Firebase (`INJAZ_FIREBASE_CONFIG`) مو سر — مصممة أصلاً تكون ظاهرة بكود أي موقع، والحماية الحقيقية تصير عبر قواعد الأمان (Rules) وتسجيل الدخول، مو عبر إخفاء الإعدادات.
- الكتابة محصورة بأجهزة سجّلت دخول مجهول عبر التطبيق — وهذا التسجيل يصير بس لما تفتح `admin.html` (لوحة التحكم)، فيها بالضبط. لوحة المتابعة (`index.html`) أبداً ما تسجّل دخول، تضل قراءة فقط دائماً.
- بما إن القراءة عامة (`read: true`)، أي شخص عنده رابط لوحة المتابعة يكدر يشوف بياناتك — تجنّب كتابة معلومات حساسة جداً بخانة الملاحظات.
- الخطة المجانية من Firebase (Spark) كافية جداً لهذا الاستخدام الشخصي، بلا أي تكلفة.
- قفل الدخول (PIN) بالإعدادات المتقدمة هو رادع إضافي بسيط بس (يمنع فتح `admin.html` نفسها من نفس جهازك)، مو تشفير حقيقي.

## 🕹️ طريقة الاستخدام

### تسجيل وقت القراءة أو الاستراحة أو النوم (طريقتين)
- ثلاث بطاقات منفصلة: **دراستي**، **استراحاتي**، **نومي** — كل وحدة تشتغل بنفس الطريقتين:
- **بالعداد:** اضغط زر البدء المناسب («ابدأ القراءة» / «ابدأ الاستراحة» / «ابدأ النوم»)، ويبدأ عداد حي يحسب الوقت حسب ساعة جهازك. لما تخلص اضغط زر الإنهاء («إنهاء الجلسة» / «صحيت 🌅»)، وتنفتح مباشرة نافذة صغيرة تكتب فيها شسويت بالضبط.
- **يدوياً:** اضغط «إضافة يدوية» تحت العداد، وحدد وقت البداية والنهاية مباشرة.
- بأي وقت تكدر تضغط أيقونة ✏️ بجنب أي جلسة لتعدّل وقتها أو تضيف/تعدّل ملاحظتها، أو 🗑️ لحذفها.
- عداد واحد بس يشتغل بنفس الوقت (ما تكدر تبدأ نوم وانت العداد شغال بالقراءة مثلاً) — لازم تخلّص الحالي أول.

### الإنجازات والنقاط
- أضف أهداف يومك بخانة «شنو تريد تنجز اليوم؟»، وأشّر عليها لما تخلصها.
- النسبة المئوية = عدد الأهداف المكتملة ÷ العدد الكلي.
- النقاط = (دقائق القراءة × نقاط/دقيقة) + (عدد الإنجازات المكتملة × نقاط/إنجاز) — تكدر تغيّر هذي المعادلة من الإعدادات.
- خط اليوم بالأعلى يعرض شكل يومك بلمحة وحدة: أخضر/ذهبي للقراءة وبرتقالي/أزرق للاستراحة، حسب النمط المختار.

### الأنماط الأربعة (⚙️ الإعدادات → المظهر)
| النمط | الفكرة |
|---|---|
| 🌙 ليلي | مصباح دافئ بغرفة هادئة، مريح للسهرات |
| 🌿 أجواء دراسية | لوح طباشير — تركيز هادئ بروح الصف |
| ☀️ طبيعي | واضح وبسيط بضوء النهار |
| 🎨 بكيفك | تختار لونك الأساسي والثانوي والقاعدة (فاتح/غامق) بنفسك |

لوحة المتابعة عندها زر 🎨 مستقل بالأعلى يخلي كل شخص يشوفها يختار النمط المريح لعينه، بدون ما يأثر على اختيارك انت.

### ⏱️ عداد لحظي عند العائلة
لما تضغط «ابدأ القراءة/الاستراحة/النوم»، أهلك يشوفون بطاقة صغيرة تنبض تقول «يقرأ الآن...» مع عداد يعد ثانية بثانية بنفس لحظة عدّك انت — يختفي لمن تضغط إنهاء. هذا التحديث يوصل فوري (بدون فترة انتظار) خصيصاً لحظة البدء والإيقاف، حتى العد يوصلهم بأقرب وقت ممكن.

### ⭐ تقييم العائلة (بلوحة المتابعة)
بأسفل لوحة المتابعة، فيه بطاقة «شاركونا رأيكم» تخلي أي واحد من العائلة يحط تقييم من 5 نجوم و/أو يكتب كلمة تشجيع — **اختياري بالكامل**، ما اكو أي إجبار. لما يضغطون «إرسال عبر واتساب» أو «نسخ الرسالة»، تتكوّن رسالة جاهزة يرسلونها لك مباشرة بأي تطبيق يحبون. هذا مو نظام تلقائي (بما إن لوحة المتابعة قراءة فقط لأسباب أمنية) — يعني ما ينحفظ أو يوصلك شي إلا إذا هم يرسلونه بأنفسهم.

### 📅 اليوم / الأسبوع / الشهر
فوق كل لوحة (التحكم والمتابعة) فيه 3 أزرار: «اليوم»، «الأسبوع»، «الشهر».
- **الأسبوع:** كل الإحصائيات تتحول لمجموع الأسبوع الحالي (الأحد للسبت)، وخط اليوم يتحول لمخطط أعمدة (سطر لكل يوم).
- **الشهر:** خريطة حرارية بأسلوب "مربعات GitHub الملونة" — كل مربع يوم بالشهر الحالي، وكل ما غمق لونه يعني قريت أكثر بذيك اليوم. مرّر فوق أي مربع تشوف تفاصيله.
- بكل الحالتين، قوائم الجلسات والإنجازات تعرض كل شي من الفترة المختارة، وكل عنصر يبين وسم صغير يقول من أي يوم بالضبط.
- **اضغط على أي سطر يوم** بمخطط الأسبوع (أو مربع بخريطة الشهر) عشان تفتح نافذة تفاصيل ذيج اليوم كاملة: كل الجلسات الثلاث والإنجازات، وتكدر تعدّل أو تحذف منها مباشرة (بلوحة التحكم).
- الهدف = هدفك اليومي × عدد أيام الفترة (أسبوعي = ×7، شهري = ×عدد أيام الشهر).
- بلوحة التحكم فقط: خيارات إضافة جلسة جديدة تختفي بوضعي الأسبوع والشهر (تخص "اليوم" حصراً)، بس تكدر تعدّل/تحذف أي جلسة أو إنجاز من أي يوم عادي.
- **مهم:** البيانات نفسها محفوظة للأبد بكل الأحوال (كل يوم بمفتاحه الخاص) — هذي الأزرار بس طرق عرض إضافية، ومو شرط لحفظ البيانات.

## 🛠️ تخصيص إضافي
- **الاسم الظاهر بالعنوان:** الإعدادات → الأهداف والنقاط → اسمك.
- **هدف القراءة اليومي:** نفس التبويب، بالساعات.
- **نسخة احتياطية:** الإعدادات → متقدم → تصدير/استيراد، يحفظ كل بياناتك بملف JSON على جهازك.
- **تغيير الخطوط أو الألوان الأساسية للأنماط:** عدّل المتغيرات بأعلى ملف `css/themes.css`.

## ❓ استكشاف الأخطاء

### «المعلومات اللي أكتبها بلوحة التحكم ما تطلع بلوحة المتابعة»
جرب بالترتيب:
1. **تأكد أساساً إنك كملت كل خطوات «تفعيل المشاركة»** بالأعلى: عدّلت `INJAZ_FIREBASE_CONFIG` بالملفين، رفعتهم على GitHub، وفعّلت Anonymous بـ Authentication.
2. افتح `admin.html` → ⚙️ الإعدادات → **المشاركة مع العائلة**، وشوف **حالة المزامنة**: لو حمراء 🔴، اضغط عليها وشوف **«تفاصيل الخطأ»** يطلع نص واضح.
3. استخدم زر **«فحص الاتصال»** — يفحص فعلياً اتصال قاعدة البيانات وتسجيل الدخول المجهول، ويقول لك بالضبط وين المشكلة (إعدادات غير صحيحة، Anonymous مو مفعّل، قواعد أمان ترفض، أو كله تمام).
4. تأكد إن قواعد الأمان (Rules) بصفحة Firebase مطابقة تماماً لما كتبناه بالخطوة 2 أعلاه (`.write: "auth != null"`)، وإن Anonymous Sign-in فعلاً مفعّل من تبويب Authentication.
5. تأكد إن نفس `INJAZ_FIREBASE_CONFIG` بالضبط موجود بالملفين `admin.html` و`index.html` (نسخة قديمة بأحدهما تكسر المزامنة).
6. مع Firebase، ما اكو تأخير تخزين مؤقت — التحديث المفروض يوصل فوري. إذا لوحة المتابعة مفتوحة من قبل، جرب تسكّرها وتفتحها من جديد مرة وحدة (لتأكيد الاتصال الحي من جديد).

### «فشلت المزامنة» أو رسائل شبكة متكررة
- تأكد من اتصال الإنترنت (جرب تبدّل بين واي فاي وبيانات الموبايل).
- تأكد إن رابط `databaseURL` بالإعدادات المنسوخة صحيح ومو ناقص أي جزء.
- تأكد إن قاعدة البيانات (Realtime Database) فعلاً مفعّلة بمشروعك بـ Firebase (خطوة 2 بالأعلى)، مو بس منشأة كمشروع فاضي.
- تأكد إن Anonymous Sign-in مفعّل (Authentication → Sign-in method) — بدونه أي محاولة كتابة ترفض.

### مشاكل ثانية
- **الصفحة بيضاء أو ما تشتغل الأزرار:** تأكد إنك رفعت مجلدي `css` و`js` كاملين (بما فيها `js/firebase-bridge.js`)، مو بس ملفات HTML.
- **لوحة المتابعة تقول «ما نشر بيانات بعد»:** لازم تسوي أول مزامنة من لوحة التحكم (اضغط «مزامنة الآن»).
- **رجعت وحدة أحدث بجهاز ثاني:** إذا فتحت لوحة التحكم بجهاز جديد وعندك مزامنة مفعّلة، راح يسألك إذا تريد تحمّل آخر نسخة محفوظة.
- **مشاكل برفع الموقع نفسه على GitHub Pages (مو المزامنة):** تأكد إن Settings → Pages مضبوط على «Deploy from a branch» (الفرع main، المجلد /root) — هذا المشروع ملفات جاهزة بلا بناء، فما يحتاج أي GitHub Actions Workflow.
